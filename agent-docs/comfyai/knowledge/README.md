# Knowledge Base

This directory contains pattern knowledge that cannot be derived from node schemas alone — idiomatic combinations, model-specific settings, hardware guidance, and workflow conventions.

If you are here because you need to know *how* to use a node (inputs, outputs, schema), that is in `nodes/` — not here. This directory is for *when* and *why*.

→ See `knowledge/index.md` to find the right entry.
