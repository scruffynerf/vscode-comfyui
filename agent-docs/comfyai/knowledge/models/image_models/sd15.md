# SD 1.5 Models

Reference for Stable Diffusion 1.5 and community checkpoints built on it (DreamShaper, Realistic Vision, Deliberate, EpicRealism, etc.).

For SDXL (the successor architecture), see [sdxl.md](sdxl.md).

---

## Architecture overview

- Native resolution: 512×512 (generates poorly above this without hires fix)
- Text encoder: single CLIP-L
- Prompt style: tag-based (short tokens, comma-separated)
- Load with `CheckpointLoaderSimple` (AIO — includes MODEL + CLIP + VAE)

---

## VAE

The bundled VAE from most SD1.5 checkpoints works fine — no swap needed unlike SDXL. Optional standalone upgrade: `vae-ft-mse-840000-ema-pruned.safetensors` — a fine-tuned VAE that produces slightly sharper decodes.

---

## Sampler settings

- Steps: 20–30
- CFG: 7.0 (range 5–10 is safe)
- Sampler: `dpmpp_2m`
- Scheduler: `karras`

---

## IP-Adapter models

Requires: ComfyUI-IPAdapter-plus custom node. Load with `IPAdapterModelLoader`.
Also needs a `CLIPVisionLoader` — see `available-models.json` → `clip_vision`.

In `available-models.json` → `ipadapter`.

| Model | Strength | Notes |
|---|---|---|
| `ip-adapter_sd15.safetensors` | Standard | General image conditioning |
| `ip-adapter_sd15_light.safetensors` | Light | Subtle influence, more text control |
| `ip-adapter_sd15_light_v11.bin` | Light v1.1 | Updated light version |
| `ip-adapter-plus_sd15.safetensors` | Strong | Stronger image fidelity |
| `ip-adapter-plus-face_sd15.safetensors` | Face-focused | Optimized for facial features |
| `ip-adapter-full-face_sd15.safetensors` | Full face | Maximum face fidelity |
| `ip-adapter_sd15_vit-G.safetensors` | ViT-G | Uses larger CLIP Vision (`clip_vision_g.safetensors`) |

**CLIP Vision**: `clip_vision_g.safetensors` (for `vit-G` variants) or `CLIP-ViT-H-14-laion2B-s32B-b79K.safetensors`.

### FaceID models

FaceID captures face identity (not just style) for consistent character generation. Also needs a companion **FaceID LoRA** in `available-models.json` → `loras`.

| Model | Notes |
|---|---|
| `ip-adapter-faceid_sd15.bin` | Standard FaceID |
| `ip-adapter-faceid-plus_sd15.bin` | FaceID + style |
| `ip-adapter-faceid-plusv2_sd15.bin` | FaceID + style v2 |
| `ip-adapter-faceid-portrait_sd15.bin` | Multiple reference images |
| `ip-adapter-faceid-portrait-v11_sd15.bin` | Portrait v1.1 |

## ControlNet models

Load with `ControlNetLoader` → `ControlNetApply` or `ControlNetApplyAdvanced`.

In `available-models.json` → `controlnet`. v1.1 fp16 LoRA-rank128 variants (compact, fast).

| Model | Guidance type |
|---|---|
| `control_lora_rank128_v11p_sd15_canny_fp16.safetensors` | Edge detection (Canny) |
| `control_lora_rank128_v11f1p_sd15_depth_fp16.safetensors` | Depth map |
| `control_lora_rank128_v11p_sd15_openpose_fp16.safetensors` | Human pose |
| `control_lora_rank128_v11p_sd15_lineart_fp16.safetensors` | Line art |
| `control_lora_rank128_v11p_sd15_normalbae_fp16.safetensors` | Normal map |
| `control_lora_rank128_v11p_sd15_mlsd_fp16.safetensors` | Straight lines (architecture) |
| `control_lora_rank128_v11e_sd15_shuffle_fp16.safetensors` | Style reference shuffle |
| `control_lora_rank128_v11e_sd15_ip2p_fp16.safetensors` | Instruction-based (pix2pix) |
| `control_lora_rank128_v11p_sd15_inpaint_fp16.safetensors` | Inpainting guidance |
| `control_lora_rank128_v11f1e_sd15_tile_fp16.safetensors` | Tile / texture |

**Preprocessors**: most require a preprocessor node (Canny edge, DPT depth, etc.). Use the ControlNet auxiliary nodes.

Also in `available-models.json` → `diff_controlnet`: older kohya-ss diff format, less commonly used.
