# Knowledge Base

Pattern knowledge that cannot be derived from node schemas alone — idiomatic combinations, model-specific settings, hardware guidance, and workflow conventions.

If you need to know *how* to use a node (inputs, outputs, schema) → `nodes/`. This directory is for *when* and *why*.

**This directory is extension-provided and wiped on reinstall.** Your notes go in `wiki/`. See `wiki/README.md`.

→ Start with `index.md` to find what you need.
